# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP:
- слой представления, отвечает за отображение данных на странице,
- слой данных, отвечает за хранение и изменение данных
- презентер, отвечает за связь представления и данных


## Типы и интерфейсы приложения

### Карточка

```
interface ICard {               – Типизация данных приходящих с сервера
  id:           string;         – id объекта
  category:     string;         – категория 
  title:        string;         – название
  image:        string;         - изображение
  description:  string;         - описание
  price:        number | null;  - цена, или null если ее нет
}
```
### Коллекция карточек
```
interface ICardModel {          – Интерфейс модели данных каталога карточек товара
  items:        ICard[];        – типизация массива для хранения объектов определенного типа
}
```
### Корзина
```
interface IBasketModel {        – Интерфейс модели данных корзины
  items:        ICard[];        – типизация массива для хранения объектов определенного типа
  totalPrice:   number;         – для подсчета общей суммы товаров, находящихся в корзине
}
```
### Заказ
```
interface IOrderModel {         – Интерфейс модели данных заказа
  payment:      TPayment;       – способо оплаты заказа
  address:      string;         – адрес доставки
  email:        string;         – электронная почта
  phone:        string;         – номер телефона
  items:        ICard[];        – типизация массива для хранения объектов определенного типа
  totalPrice:   number;         – для подсчета общей суммы товаров, находящихся в заказе
}
```
### Ответ сервера
```
interface OrderResponse {       – Ответ сервера на заказ
  id:           string;         – id совершенного заказа
  totalPrice:   number;         – указание полной стоимости заказа
}
```
### Способы оплаты
```
enum TPayment {                 – Способы оплаты
  CASH =        'cash',         – наличка
  CARD =        'card'          – картой(онлайн)
}
```

## Базовый код

### Класс Api

В работе приложения будут обращения к серверу, для получения данных, для отправки заказа. Для этого в базовом коде есть класс Api. В полях класса базовый url адрес и опции запроса. В конструкторе, при создании экземпляра класса передаются базовый url и опции запроса.

Методы get и post служат для отправки запросов на получение и публикацию данных. 

Есть метод для обработки ответов сервера – handleResponse, если ответ пришел и нет ошибок данные передаются для работы дальше, если есть ошибка то выбрасывается ошибка с указанием причины

### Класс Component

Абстрактный родительский класс для всех классов слоя представления, в конструктор принимает container. Методы:
  toggleClass()                 – добавление/удаление классов элементам разметки
  setText()                     – установка текстового содержимого элемента разметки
  setDisabled()                 – смена статуса блокировки элемента разметки 
  setHidden()                   – скрыть элемент 
  setVisible()                  – показать элемент
  setImage()                    – установка изображения элемента
  render()                      – функция для возрата разметки элемента, для последующей отрисовки в пользовательском интерфейсе

### Класс EventEmitter

Приложение построенно на событийно-оринтированной парадигме. Класс EventEmitter регулирует работу генерации событий в экземплярах слоя отображения и слоя данных. Класс реагирует на события при изменениях в слоях и генерирует новую разметку или изменение в данных.

Основные методы класс используемые в приложении:

emit()                          – генерация события 
on()                            – подписка на событие 

### Класс Model

Абстрактный класс, является родительским классом для всех классов слоя данных. В конструктор принимает объект, и объект класса EventEmitter.

Метод emitChanges()             – сообщает всем, что модель хранения данных изменилась


## Слой данных (Model)

#### Класс CardModel 

Класс отвечает за хранение данных и логику работы с данными карточек пришедших ответом с сервера. \
У экземпляра класса есть три поля:
- _items: ICard[]               - массив карточек объектов
- _preview: string | null       - id карточки, выбранной для просмотра в модальном окне
- events: IEvents               - экземпляр класс `EventEmitter` для генерации событий при изменении данных

У класса есть методы, для работы с этими данными:
- set items(data: ICard[]): void устанавливает полученные данные в поле класса _items и генерирует событие ITEMS_UPDATED
- get items(): ICard[] возращает установленный в поле _items массив объектов ICard, например это будет полезно при работе с отображением
- getItem(id: string): ICard | undefined метод для работы с модальным окном, чтобы выводить подробную информацию по выбранному товару
- set preview(id: string) установка объекта типа ICard в поле preview для отображения в модальном окне

#### Класс OrderModel

Класс хранит информацию из полей форм для формирования заказа и отправки данных на сервер. \
Для хранения данных есть несколько полей:

- payment: TPayment             -  для указания способа оплаты _онлайн_ или _наличкой_
- address: string               - для указания адреса покупателя
- email: string                 - для указания почты 
- phone: string                 - поле для номера телефона покупателя
- items: ICard[]                - выбранные товары для заказа
- totalPrice: number | null     - итоговая сумма

Методы в классе представлены сеттерами и геттерами для получения и установки значения и один для подсчета общей стоимости заказа


#### Класс BasketModel

Для хранения товаров который пользователь положит в корзину. Показывает список товаров, их цену, и подсчитывает общую сумму заказа.

Поля: 
- items: ICard[]                - здесь хранятся товары добавленные в корзину
- totalPrice                    - итоговая сумма всех товаров в корзине

Помимо геттеров и сеттеров есть методы для работы с экземпляром класса:
- addToBasket(item: ICard)      - принимает на вход объект типа ICard и помещает его в поле массив items и генерирует событием ITEM_ADDED
- removeFromBasket()            - специальный метод, для механики удаления товара из корзины, генерирует событие ITEM_REMOVED
- clearBasket                   - метод, который используется когда заказ сформирован, очищает корзину (поле экземпляра items), генерирует событие BASKET_CLEARED
 

## Слой представление (View)

Все классы слоя представления отвечают за отображение внутри контейнера (DOM-элемента) передаваемых в него данных

#### Класс Modal

Единственный класс для работы с модальным окном, представляет из себя пустой контейнер, куда будет передаваться разный контент в зависимости от действий пользователя, это может быть корзина, ввод контактов, адрес доставки, а также ответ об успешной оплате заказа. 

Для этого в полях экземпляра есть:

_content: HTMLElement           - для передачи разметки контента в контейнер для отображения
closeButton: HTMLButton         - для обработки событий клика и генерации события, к примеру закрытия
formElement?: HTMLFormElement   - для передачи разметки формы и работы с ней, потребуется на этапе работы с формой оплаты и ввода почты и номера
inputs?: NodeListOf<HTMLInputElement> - находим все элементы ввода из формы (formElement)
errors?: Record<string, HTMLElement> - объект хранящий все элементы для вывода ошибок под полями формы с привязкой к атрибуту name инпутов
actionButton: HTMLButton        - специальная кнопка для перехода к следующему шагу заказа. Переход от корзины к вводу адреса, email и номера телефона 

Констуктор: при создании передается разметка HTML элемента-контейнера и экземпляр events: EventEmitter (для генерации событий). В конструкторе добавляется слушатель на поле кнопки, и в обработчике события генерируется событией MODAL_CLOSE

методы экзмпляра close/open для работы с открытием и закрытием модального окна. Внутри в щзависимости от ситуации чистится контент и добаляется новый. \
некоторые методы будут использоваться из родительского класса Component для ограничения `actionButton` в случае если введеные данные не валидны или не введены вовсе. \
showError()/hideError()         - методы для того чтобы отобразить или скрыть ошибку

модальное окно можно закрыть кликом по оверлею, нажатием на клавишу ESC или на крестик в правом верхнем углу. Также модальное окно должно блокировать проукрутку основной страницы поверх которого оно размещается. 

#### Класс Card 

Класс для работы с отображением объектов типа ICard в интерфейсе. Отображаются на главной странице, в общем каталоге (ограниченная информация), в превью(полная информация), и отображение в виде строки в корзине в списке товаров. 

поля:
- id: string                    - техническое поле, для сохранения и передачи id товара
- catagory: HTMLElement         - поле для вывода информации о категории товара
- title: HTMLElement            - для вывода названия товара
- image: HTMLImageElement       - вывод изображения товара
- description: HTMLElement      - описание товара
- price: HTMLElement            - цена товара

В конструкторе принимается контейнер в котором нужно найти элементы разметки для выгрузки в них данных из объекта типа ICard. А также экземпляр класса EventEmitter для генерации событий 

из методов set/get методы для того чтобы установить или взять данные из полями

#### Класс Page

Класс для работы с элементом разметки главной страницы. \
В конструкторе принимает контейнер, из которого находим основные элементы разметки и устанавливаем их в поля этого класса.

Поля:
- _itemsContainer: HTMLElement; - для вывода карточек товаров на главную
- _basketButton: HTMLButtonElement; - кнопка корзины, на ней висит обработчик события клика для генерации события BASKET_OPEN
- _basketCount: HTMLElement;    - для отображение количества в штуках товара в корзине

для работы и управления данными в классе есть сеттеры и геттеры

### Класс Basket 

Класс для работы с корзиной в модальном окне. В конструкторе получает элемент контейнер модального окна, для вывода информации. Поля:

- basketTemplate: HTMLTemplateElement;  - установка шаблона корзины 
- basketContentElement: HTMLElement;    - установка контента шаблона для выгрузки данных
- basketOrderButton: HTMLButtonElement; - кнопка "Оформить" для генерации события и закрытия корзины и открытия следующего модального окна 

из методов сеттеры и геттеры


## Слой презентера (Presenter)

В данном приложении слой презентера не будет выделяться в отдельный слой, будет представлен в основном скрипте приложения. Малое количество кода в связи с этим подписка на события и работа по взаимодействию слоев данных и отображения будет все в одном основном скрипте, на старте приложения создается один экземпляр класс EventEmitter, он передается во все объекты классов слоев отображения и также данных.

## События, возникающие при работе с приложением

Список событий который будет использоваться в приложении:

  ITEMS_UPDATED             = 'ITEMS_UPDATED'         - загрузка всех товаров с сервера \
  MODAL_CLOSE               = 'MODAL_CLOSE'           - закрытие модального окна \
  BASKET_OPEN               = 'BASKET_OPEN'           - открытие модалки с корзиной \
  ITEM_REMOVED              = 'ITEM_REMOVED'          - удаление товара из корзины \
  ITEM_ADDED                = 'ITEM_ADDED'            - товар добавлен в корзину \
  BASKET_CLEARED            = 'BASKET_CLEARED'        - очистка корзины \
  PREVIEW_UPDATED           = 'PREVIEW_UPDATED'       - модалка с полным описание товара \
  ADDRESS_FORM_OPEN         = 'ORDER_ADDRESS_OPEN'    - модалка с адресом открыта \
  CONTACTS_FORM_OPEN        = 'ORDER_CONTACTS_OPEN'   - модалка с контактами открыта \
  ORDER_RESULT_OPEN         = 'ORDER_FINISH_OPEN'     - модалка с результатом заказа открыта \
